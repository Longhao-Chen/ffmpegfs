11 Prerequisites
12 -------------
13 
14 FFmpegfs uses FFmpeg lib for decoding/encoding. It requires the following
15 libraries:
16 
17 * gcc and g++ compilers
18 
19 * fuse (>= 2.6.0)
20 * sqlite3 (>= 3.7.13)
21 
22 * libavutil (>= 54.3.0)
23 * libavcodec (>= 56.1.0)
24 * libavformat (>= 56.1.0)
25 * libavfilter (>= 5.40.0)
26 * libswscale (>= 3.0.0)
27 
28 One of these is required, preferably libswresample as libavresample is deprecated.
29 It will be removed, but as for now is required if Libav is used (which has no
30 libswresample).
31 
32 * libswresample (>= 1.0.0)
33 * libavresample (>= 2.1.0)
34 
35 For optional DVD support you need the following libraries
36 
37 * libdvdread (>= 5.0.0)
38 * libdvdnav (>= 5.0.0)
39 
40 For optional Bluray support you need the following library
41 
42 * libbluray (>= 0.6.2)
43 
44 The commands to install just the first prerequisites follow.
45 
46 Please read the "Supported Linux Distributions" chapter in README.md
47 for details.

156 Installation
157 ------------
158 
159 FFmpegfs uses the GNU build system. If you are installing from git, you'll
160 need to first run:
161 
162  ./autogen.sh
163 
164 If you are downloading a release, this has already been done for you. To
165 build and install, run:
166 
167  ./configure
168  make
169  make install
170 
171 To build and run the check suite, do:
172 
173  make checks
174 
175 This will test audio conversion, tagging and size prediction.
176 
177 NOTE: Image embedding is not yet implemented. The test has been
178 disabled at the moment.

190 Trouble Shooting
191 ----------------
192 
193 **Mounting via /etc/fstab fails:**
194 
195 An fstab line with fuse.ffmpegfs file system fails with a strange message when
196 attempted to mount with "mount -av". No log entries, no other hints...
197 
198  mount: wrong fs type, bad option, bad superblock on /mnt/sdf1/mp3base1,
199  missing codepage or helper program, or other error
200 
201  In some cases useful info is found in syslog - try
202  dmesg | tail or so.
203 
204 Fuse is missing! Do...
205 
206  apt-get install fuse
207 
208 **fuse: failed to exec fusermount: No such file or directory:**
209 
210 Trying to mount as other than root, the message
211 
212  fuse: failed to exec fusermount: No such file or directory
213 
214 is printed.
215 
216 Fuse is missing! Do...
217 
218  apt-get install fuse
219 
220 **"ERROR: libmp3lame >= 3.98.3 not found":**
221 
222 If you run into this "ERROR: libmp3lame >= 3.98.3 not found" although you have built
223 and installed libmp3lame you may find a solution here:
224 https://stackoverflow.com/questions/35937403/error-libmp3lame-3-98-3-not-found
225 
226 **autogen.sh displays "possibly undefined macro":**
227 
228  Running autoreconf --install
229  configure.ac:46: error: possibly undefined macro: AC_DEFINE
230  If this token and others are legitimate, please use m4_pattern_allow.
231  See the Autoconf documentation.
232  autoreconf: /usr/bin/autoconf failed with exit status: 1
233 
234 You are probably missing out on pkg-config, either it is not installed or
235 not in path. "apt-get install pkg-config" (on Debian or equivalent on other
236 Linux distributions) should help.
237 
238 **If the videotag.php script does not work under PHP7**
239 
240 The script runs fine under PHP5, but when upgrading to PHP7 (or using PHP7)
241 it suddenly stops showing the list of files.
242 
243 Check the Apache2 error.log, you might see this:
244 
245 "PHP Fatal error: Uncaught Error: Call to undefined function utf8_encode() in index.php"
246 
247 This is because for some reason utf8_encode() has been moved to the XML
248 library. Just do (or similar):
249 
250  apt-get install php7.0-xml
251  systemctl restart apache2
252 
253 And your troubles should be gone.
254 
255 **"make check": all audio checks fail**
256 
257 Logfiles contain "bc: command not found", so the command line communicator is
258 missing.
259 
260 Fix by installing it (or similar):
261 
262  apt-get install bc
263 
264 **Songs get cut short**
265 
266 If songs do not play to the very end and you are using SAMBA or NFS you're in trouble.
267 
268 Happens when the files are transcoded on the fly, but never when file comes from
269 cache. This is because the result is never exactly what was predicted.
270 
271 SAMBA fills files with zeroes if the result is smaller, or cuts off the rest if the
272 file ist larger than predicted.
273 
274 NFS arbitrarily sends the correct file, or one that is cut or padded like SAMBA.
275 This can be repeated as many times as one wants to - once the file is OK, once
276 not.
277 
278 As of yet there seems to be no way around that. Maybe NFS or SAMBA can be configured
279 to cope with that, but how to is unknown to me.
280 
281 **Make reports "/bin/sh: a2x: command not found"**
282 
283 You are missing out on asciidoc, to install do (or similar):
284 
285  apt-get install asciidoc
286 
287 That should fix it. You may just install asciidoc-base to safe disk space.
288 
289 **libbluray fails to load libbluray.jar**
290 
291 When you see this message accessing blurays:
292 
293  bdj.c:340: libbluray-j2se-0.9.3.jar not found.
294  bdj.c:466: BD-J check: Failed to load libbluray.jar
295 
296 To get rid of this message simply install "libbluray-bdj", this will make it go away.
297 Though not necessary, as to read the bluray tracks java support is not required, so
298 this is simply cosmetical.
