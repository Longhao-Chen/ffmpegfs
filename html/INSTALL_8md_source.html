4 Installation from repository
5 ============================
6 
7 Debian Buster
8 -------------
9 
10 FFmpegfs 1.98 can be installed on Debian 10 Buster from Buster Backports and
11 sid (which is not recommended and therefore not described here).
12 
13 To enable Buster Backports:
14 
15  echo "deb http://deb.debian.org/debian buster-backports main" | sudo tee /etc/apt/sources.list.d/backports.list
16 
17  sudo apt-get update
18 
19 Then install FFmpegfs:
20 
21  sudo apt-get -t buster-backports install ffmpegfs
22 
23 Debian Bullseye
24 ---------------
25 
26 FFmpegfs 1.98 has been added to Debian 11 Bullseye so it is available as
27 binary distribution.
28 
29 On Debian 11 Bullseye you can simply do
30 
31  apt-get install ffmpegfs
32 
33 For Ubuntu 20.04 and Linux distributions based on it this is
34 
35  apt install ffmpegfs

37 Other Distributions
38 -------------------

44 Building FFmpegfs yourself
45 ==========================
46 
47 Prerequisites
48 -------------
49 
50 FFmpegfs uses FFmpeg lib for decoding/encoding. It requires the following
51 libraries:
52 
53 * gcc and g++ compilers
54 
55 * fuse (>= 2.6.0)
56 * sqlite3 (>= 3.7.13)
57 
58 * libavutil (>= 54.3.0)
59 * libavcodec (>= 56.1.0)
60 * libavformat (>= 56.1.0)
61 * libavfilter (>= 5.40.0)
62 * libswscale (>= 3.0.0)
63 
64 One of these is required, preferably libswresample as libavresample is deprecated.
65 It will be removed some day from FFmpeg API.
66 
67 * libswresample (>= 1.0.0)
68 * libavresample (>= 2.1.0)
69 
70 For optional DVD support you need the following libraries
71 
72 * libdvdread (>= 5.0.0)
73 * libdvdnav (>= 5.0.0)
74 
75 For optional Bluray support you need the following library
76 
77 * libbluray (>= 0.6.2)
78 
79 The commands to install just the first prerequisites follow.
80 
81 Please read the "Supported Linux Distributions" chapter in README.md
82 for details.
83 
84 **On Debian:**
85 
86  apt-get install gcc g++ make pkg-config asciidoc-base w3m
87 
88  apt-get install fuse libfuse-dev libsqlite3-dev libavcodec-dev libavformat-dev libswresample-dev libavutil-dev libswscale-dev libavfilter-dev
89 
90 To get DVD support:
91 
92  apt-get install libdvdread-dev libdvdnav-dev
93 
94 To get Bluray support:
95 
96  apt-get install libbluray-dev
97 
98 To "make doxy" (build Doxygen documentation):
99 
100  apt-get install doxygen graphviz curl
101 
102 To "make check" (run test suite):
103 
104  apt-get install libchromaprint-dev bc
105 
106 **On Suse** (please read notes before continuing):
107 
108  zypper install gcc gcc-c++
109 
110  zypper install fuse fuse-devel libsqlite3-devel libavcodec-devel libavformat-devel libswresample-devel libavutil-devel libswscale-devel
111 
112 To get DVD support:
113 
114  zypper install libdvdread-devel libdvdnav-devel
115 
116 To get Bluray support:
117

120 Suse includes non-proprietary codecs with FFmpeg only, namely mp3, AAC and H264

147 **On Funtoo Linux:**
148 
149 To get fuse suppprt and chromaprint (for make check):
150

154 To get FFmpeg with H264 etc. support, specify some "USE flags" when doing emerge:
155

157

161

169 Build FFmpeg with optimisations
170 -------------------------------
171 
172 The precompiled package of FFmpeg available for Debian, Ubuntu etc. is built
173 with common options so that it can run on many processors. To leverage the
174 full potential it may be useful to build it with optimisation options for the
175 target CPU. The resulting binaries may not run on other computers.
176 
177 FFmpeg must be built with at least libx264, libfdk_aac and libmp3lame support.
178 Other libraries, e.g. ogg, Windows Media or FLAC must be added when these
179 formats should be used as source.
180 
181 **Attention!** Remember to do "apt remove ffmpeg" before proceeding with the next
182 steps to avoid malefic blends of custom and official distribution binaries.
183 
184 For a minimum build that contains all the libraries required by FFmpegfs and
185 SSL support for convenience use:
186 
187  configure \
188  --extra-cflags='-march=native' \
189  --extra-cflags=-Ofast \
190  --enable-runtime-cpudetect \
191  --disable-static \
192  --enable-gpl \
193  --enable-shared \
194  --enable-version3 \
195  --enable-nonfree \
196  --enable-pthreads \
197  --enable-postproc \
198  --enable-openssl \
199  --enable-libvpx \
200  --enable-libvorbis \
201  --enable-avresample \
202  --enable-libmp3lame \
203  --enable-libtheora \
204  --enable-libxvid \
205  --enable-libx264
206 
207 Fix the complaints by configure, i.e. install the required
208 development packages, then
209 
210  make install
211 
212 This works for me. Decoding runs much faster with these settings.
213 
214 **NOTE:** Depending on the source formats you have it may be required
215 to add additional libraries.

217 Building source code
218 --------------------
219 
220 Download a release archive:
221 
222  wget https://github.com/nschlia/ffmpegfs/releases/download/v1.98/ffmpegfs-1.98.tar.gz
223 
224 You may check https://github.com/nschlia/ffmpegfs/releases to see if there are
225 newer releases available.
226 
227 Then unpack and cd to the source directory. To build and install, run:
228 
229  ./configure
230  make
231  make install
232 
233 To build and run the check suite, do:
234 
235  make checks
236 
237 This will test audio conversion, tagging and size prediction.
238 
239 NOTE: Image embedding is not yet implemented. The test has been
240 disabled at the moment.

244

250 * autoconf
251 * automake
252 * asciidoc (or at least asciidoc-base to save disk space)

256 
257  apt-get install gcc g++ make pkg-config autoconf asciidoc-base w3m libchromaprint-dev bc doxygen graphviz

260 
261  ./autogen.sh

265 
266  ./configure

284 Trouble Shooting
285 ----------------
286 
287 **Mounting via /etc/fstab fails:**
288 
289 An fstab line with fuse.ffmpegfs file system fails with a strange message when
290 attempted to mount with "mount -av". No log entries, no other hints...
291 
292  mount: wrong fs type, bad option, bad superblock on /mnt/sdf1/mp3base1,
293  missing codepage or helper program, or other error
294 
295  In some cases useful info is found in syslog - try
296  dmesg | tail or so.
297 
298 Fuse is missing! Do...
299 
300  apt-get install fuse
301 
302 **fuse: failed to exec fusermount: No such file or directory:**
303 
304 Trying to mount as other than root, the message
305 
306  fuse: failed to exec fusermount: No such file or directory
307 
308 is printed.
309 
310 Fuse is missing! Do...
311 
312  apt-get install fuse
313 
314 **"ERROR: libmp3lame >= 3.98.3 not found":**
315 
316 If you run into this "ERROR: libmp3lame >= 3.98.3 not found" although you have built
317 and installed libmp3lame you may find a solution here:
318 https://stackoverflow.com/questions/35937403/error-libmp3lame-3-98-3-not-found
319 
320 **autogen.sh displays "possibly undefined macro":**
321 
322  Running autoreconf --install
323  configure.ac:46: error: possibly undefined macro: AC_DEFINE
324  If this token and others are legitimate, please use m4_pattern_allow.
325  See the Autoconf documentation.
326  autoreconf: /usr/bin/autoconf failed with exit status: 1
327 
328 You are probably missing out on pkg-config, either it is not installed or
329 not in path. "apt-get install pkg-config" (on Debian or equivalent on other
330 Linux distributions) should help.
331 
332 **If the videotag.php script does not work under PHP7**
333 
334 The script runs fine under PHP5, but when upgrading to PHP7 (or using PHP7)
335 it suddenly stops showing the list of files.
336 
337 Check the Apache2 error.log, you might see this:
338 
339 "PHP Fatal error: Uncaught Error: Call to undefined function utf8_encode() in index.php"
340 
341 This is because for some reason utf8_encode() has been moved to the XML
342 library. Just do (or similar):
343 
344  apt-get install php7.0-xml
345  systemctl restart apache2
346 
347 And your troubles should be gone.
348 
349 **"make check": all audio checks fail**
350 
351 Logfiles contain "bc: command not found", so the command line communicator is
352 missing.
353 
354 Fix by installing it (or similar):
355 
356  apt-get install bc
357 
358 **Songs get cut short**
359 
360 If songs do not play to the very end and you are using SAMBA or NFS you're in trouble.
361 
362 Happens when the files are transcoded on the fly, but never when file comes from
363 cache. This is because the result is never exactly what was predicted.
364 
365 SAMBA fills files with zeroes if the result is smaller, or cuts off the rest if the
366 file ist larger than predicted.
367 
368 NFS arbitrarily sends the correct file, or one that is cut or padded like SAMBA.
369 This can be repeated as many times as one wants to - once the file is OK, once
370 not.
371 
372 As of yet there seems to be no way around that. Maybe NFS or SAMBA can be configured
373 to cope with that, but how to is unknown to me.
374 
375 **Make reports "/bin/sh: a2x: command not found"**
376 
377 You are missing out on asciidoc, to install do (or similar):
378 
379  apt-get install asciidoc
380 
381 That should fix it. You may just install asciidoc-base to safe disk space.
382 
383 **libbluray fails to load libbluray.jar**
384 
385 When you see this message accessing blurays:
386 
387  bdj.c:340: libbluray-j2se-0.9.3.jar not found.
388  bdj.c:466: BD-J check: Failed to load libbluray.jar
389 
390 To get rid of this message simply install "libbluray-bdj", this will make it go away.
391 Though not necessary, as to read the bluray tracks java support is not required, so
392 this is simply cosmetical.
393 
394 **Switching between repository version and source builds**
395 
396 It is easy to switch between both worlds. You can do that as many
397 times as you want to, alas the cache directory will be cleared
398 every time. But it will be rebuild in the background so this will
399 almost go unnoticed.
400 
401 To switch from repository to a source build do
402 
403  apt-get remove ffmpegfs
404 
405 Then follow the steps under "Building FFmpegfs yourself". If you do
406 not remove the repository version, you self-build could be unadvertedly
407 up- or downgraded when a new version becomes available from the repository.
408 
409 To switch from a source build to a repository installation change to
410 the build directory and do
411 
412  make uninstall
413 
414 Then follow the steps under "Installation from repository".
